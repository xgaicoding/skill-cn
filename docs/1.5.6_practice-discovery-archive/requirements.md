# v1.5.6 实践模式可发现性优化 需求文档

> 版本：1.5.6 | 日期：2026-02-13 | 状态：需求评审

---

## 背景

实践模式（v1.1.0 上线）是 Skill Hub 的核心差异化功能，但上线 17 天后发现：

1. **用户可能没注意到实践模式**：ModeDock 在右侧，首次引导动画可能被忽略
2. **价值主张不够清晰**："实践模式"这个名字太抽象，用户不知道能得到什么
3. **缺少数据验证**：不知道多少用户发现了实践模式，使用率如何

v1.5.6 聚焦解决**可发现性问题**，让更多用户知道并使用实践模式，为后续优化提供数据支持。

---

## 核心目标

1. **提升实践模式的可发现性**：让首次访问用户在 10 秒内注意到实践模式
2. **优化价值主张**：让用户一眼看懂实践模式的价值（"看案例找方案"）
3. **建立数据基础**：埋点追踪用户行为，验证优化效果

---

## 需求清单

### 一、首页引导优化

#### 1.1 首次访问引导 Banner

**触发条件**：
- 用户首次访问首页（localStorage 标记）
- 仅在桌面端显示（移动端空间有限，暂不做）

**位置**：
- 首页顶部，Header 下方
- 不遮挡主内容，可关闭

**内容**：
```
┌─────────────────────────────────────────────────────────┐
│  💡 发现新功能：实践模式                          [✕]  │
│  不只是 Skills 目录，更有真实案例和解决方案            │
│  [立即体验 →]                                           │
└─────────────────────────────────────────────────────────┘
```

**交互**：
- 点击"立即体验"：自动切换到实践模式，关闭 Banner
- 点击 [✕]：关闭 Banner，记录到 localStorage（不再显示）
- Banner 显示 3 秒后自动淡出（用户可以在淡出前点击）

**视觉**：
- 浅色背景 + 渐变边框（与品牌色一致）
- 不要太抢眼，避免打断用户浏览

---

#### 1.2 ModeDock 视觉优化

**现状问题**：
- ModeDock 在右侧，首次引导动画可能被忽略
- "实践模式"/"Skill 模式"文案太抽象

**优化方案**：

1. **文案优化**：
   - "实践模式" → "看案例"
   - "Skill 模式" → "找工具"
   - 更直接，用户一眼看懂

2. **视觉强化**（可选，需设计评审）：
   - 首次引导动画改为"呼吸光效"（更明显）
   - 或者在 ModeDock 旁边加一个小箭头提示："试试这个 →"

3. **位置调整**（可选，需用户测试）：
   - 考虑将 ModeDock 移到左侧（更符合阅读习惯）
   - 或者改为顶部 Tab 切换（更传统，但可能不够酷）

**本期范围**：
- ✅ 文案优化（立刻可做）
- ⚠️ 视觉强化（需设计评审）
- ❌ 位置调整（留到 v1.6+ 根据数据决定）

---

### 二、实践模式价值主张优化

#### 2.1 首屏空状态优化

**现状问题**：
- 如果实践案例数量少，首屏可能显得空
- 用户不知道实践模式的价值

**优化方案**：
- 在实践模式首屏顶部加一行说明文案：
  ```
  💡 真实案例 · 实战方案 · 快速上手
  ```
- 如果实践案例 < 6 个，显示引导卡片：
  ```
  ┌─────────────────────────────────────┐
  │  📝 实践案例持续更新中...           │
  │  想分享你的实战经验？               │
  │  [投稿实践案例 →]                   │
  └─────────────────────────────────────┘
  ```

---

#### 2.2 实践卡片视觉优化（可选）

**现状**：
- 实践卡片已经做得很好（定高、渐隐、hover 浮层）
- 但可以考虑加一些视觉元素强化"案例"属性

**可选优化**：
- 卡片左上角加一个小标签："实战案例"
- 或者在卡片顶部加一个细线条，区分"案例"和"工具"

**本期范围**：
- ❌ 暂不做（避免过度设计）
- 留到 v1.6+ 根据用户反馈决定

### 三、数据埋点

#### 3.1 核心指标

| 指标 | 说明 | 埋点位置 |
|------|------|----------|
| **实践模式发现率** | 访问首页的用户中，多少人切换到实践模式 | ModeDock 点击事件 |
| **实践模式使用率** | 切换到实践模式的用户中，多少人有后续操作（滚动、点击卡片） | 实践模式页面滚动、卡片点击 |
| **引导 Banner 转化率** | 看到 Banner 的用户中，多少人点击"立即体验" | Banner 展示、点击事件 |
| **模式停留时长** | 用户在实践模式 vs Skills 模式的平均停留时长 | 页面停留时长（按模式区分） |
| **实践案例点击率** | 实践卡片的点击率（跳转原文） | 卡片"跳转原文"按钮点击 |

#### 3.2 埋点方案

**工具选择**：
- 使用现有的 Vercel Analytics（已集成）
- 或者 Supabase Analytics（免费，易集成）

**埋点事件**：
```javascript
// 模式切换
track('mode_switch', { from: 'skills', to: 'practices' })

// Banner 展示
track('banner_show', { type: 'practice_discovery' })

// Banner 点击
track('banner_click', { type: 'practice_discovery', action: 'try_now' })

// 实践卡片点击
track('practice_card_click', { practice_id: 123, action: 'view_original' })

// 页面停留时长
track('page_view_duration', { mode: 'practices', duration_seconds: 45 })
```

### 四、A/B 测试（可选）

**测试假设**：
- 首页默认实践模式 vs 默认 Skills 模式，哪个留存和转化更好？

**测试方案**：
- 50% 用户看到默认实践模式
- 50% 用户看到默认 Skills 模式（现状）
- 追踪 7 天留存、页面停留时长、跳出率

**本期范围**：
- ❌ 暂不做 A/B 测试（需要更多流量和时间）
- 先做埋点，收集 1-2 周数据后再决定是否测试

## 成功指标（上线后 2 周）

| 指标 | 目标 | 现状（估计） |
|------|------|--------------|
| 实践模式发现率 | > 40% | < 20%（估计） |
| 实践模式使用率 | > 60% | 未知 |
| 引导 Banner 转化率 | > 30% | - |
| 实践案例点击率 | > 15% | 未知 |

**验证方式**：
- 上线后 2 周收集数据
- 如果实践模式发现率 > 40%，说明优化有效
- 如果使用率 > 60%，说明用户认可实践模式的价值
- 根据数据决定下一步优化方向（是否改默认模式、是否调整 ModeDock 位置等）
