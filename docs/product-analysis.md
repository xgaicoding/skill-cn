# Skill Hub 中国 — 产品深度分析报告

> 由产品经理视角和市场经理视角并行分析，覆盖 UX、功能、信息架构、增长、社区、商业化六大维度
>
> 分析日期：2026-02-07

---

## 一、用户体验 (UX) — 产品经理视角

### 1.1 用户旅程断裂

- **新用户无引导**：首次访问直接面对 Skill 卡片，不理解"Skill 是什么、怎么用、有什么价值"。唯一的入门教程指向飞书外链，跳出率高。
- **下载后无引导**：用户下载 ZIP 后旅程就断了，没有"下一步做什么"的安装指引。
- **Skill ↔ 实践关系不直观**：两个模式平行展示，缺乏上下文引导。

**建议**：

- 增加 3~4 步轻量首次引导（存 localStorage 只展示一次）
- 下载成功后展示安装指引浮层
- Hero 区增加演示 GIF

### 1.2 搜索与发现体验弱

- 搜索仅为 `ILIKE` 模糊匹配，不支持同义词、拼写容错、中文分词
- 标签只有 5 个一级分类（研究/编程/写作/数据分析/设计/生产力），过于粗粒度且互斥
- 无个性化推荐，所有用户看到完全相同的列表

**建议**：

- 短期：支持多标签筛选 + 二级分类
- 中期：引入 Supabase FTS 全文搜索
- 长期：基于用户行为做个性化推荐

### 1.3 详情页问题

- **移动端不展示 SKILL.md 文档内容**：移动端用户无法阅读使用说明
- **移动端两个核心按钮都不可用**（下载 ZIP / 投稿实践 → 弹 toast "请前往 PC 版"）
- PC 端 SKILL.md 和热门实践用 Tab 切换，文档被折叠，发现成本高
- 仓库信息不够丰富（缺少 License、主要语言等）

**建议**：

- 移动端增加 Markdown 文档展示（可默认折叠）
- 移动端改为"复制 npx 命令"
- 增加面包屑导航

### 1.4 加载与交互

- 详情页 GitHub 同步无视觉反馈（后台刷新用户不知情）
- 搜索是表单提交整页刷新，无实时搜索体验
- 错误信息过于技术化

**建议**：

- 同步时增加微小指示器
- 搜索改为 300ms debounce 输入即搜
- 错误信息用户友好化

## 二、功能完整性 — 产品经理视角

### 2.1 核心缺失

| 缺失功能           | 影响                                           |
| -------------------- | ------------------------------------------------ |
| **收藏/书签**      | 用户无法"保存以后看"，已有 OAuth 但登录后无功能 |
| **版本管理/更新通知** | 下载后不知道仓库是否更新                       |
| **评价/评分系统**  | 质量完全依赖 heat_score，缺少用户直接反馈      |
| **移动端下载**     | 移动端用户只能"看"不能"用"                     |

### 2.2 现有功能深度不够

- **实践文章只能跳转原文**，无站内摘要阅读，无法积累内容资产
- **技能包只加了个标签**，不展示包含哪些子技能
- **下载次数数据已有但前端未展示**（好的社会证明信号被浪费）
- **热度算法失衡**：1 个实践 ≈ 6666 个 Star，可能导致排名失真

## 三、获客与传播 — 市场经理视角

### 3.1 SEO 严重缺失（P0 级问题）

| 问题                     | 现状                               |
| -------------------------- | ------------------------------------ |
| 无 `sitemap.xml`         | 搜索引擎无法高效发现页面           |
| 无 `robots.txt`          | 爬取规则不明确                     |
| 无 Open Graph / Twitter Card | 社交分享只显示纯文字链接       |
| 详情页无独立 SEO 元数据 | 所有页面共用首页 title/description |
| 首页纯客户端渲染         | 搜索引擎抓到的是空壳             |

**建议**：

- 新增 `app/sitemap.ts` + `app/robots.ts`
- 为详情页添加 `generateMetadata`
- 首屏改为 SSR 预渲染

### 3.2 社交传播机制为零

- 整个产品没有分享按钮
- URL 是唯一传播手段，但分享出去无 OG 图预览
- 实践文章直接外跳，平台零流量截留

**建议**：

- 详情页 + 卡片增加"复制链接 / 分享到微信"按钮
- 移动端用 Web Share API
- 配置动态 OG 图

### 3.3 渠道布局单一

唯一社区入口是微信群二维码（且有过期风险），没有公众号、知乎专栏、即刻、RSS 等多渠道覆盖。

## 四、增长引擎 (AARRR) — 市场经理视角

| 环节                     | 现状评分 | 关键问题                                                 |
| -------------------------- | ---------- | ---------------------------------------------------------- |
| **Acquisition（获客）**  | ⭐       | 几乎完全依赖口碑，SEO/社交/内容分发全缺                 |
| **Activation（激活）**   | ⭐⭐     | 新用户不理解 Skill 概念；仅 GitHub OAuth，非开发者门槛高 |
| **Retention（留存）**    | ⭐       | 无收藏、无个人中心、无推送通知、无回访理由               |
| **Revenue（变现）**      | —        | 完全免费，无商业化设计                                   |
| **Referral（裂变）**     | ⭐       | 无分享按钮、无邀请机制、投稿门槛高（GitHub Issue）       |

**关键改进**：

- 支持微信登录（中国市场几乎必备）
- 增加"一键试用"（直接展示 Prompt 内容，可复制到 Claude 使用）
- 增加场景化推荐（"写代码用这个""做数据分析用这个"）

## 五、社区与生态

### 现状

- 社区仅有微信群二维码，讨论无法结构化沉淀
- UGC 全部通过 GitHub Issue 提交，非开发者根本不会操作
- 没有评论、评分、创作者主页、排行榜等社区基础设施

### 建议

- 站内做轻量投稿表单（替代 GitHub Issue），大幅降低贡献门槛
- 增加 Skill 详情页"好用/不好用"投票
- 建设创作者 Profile 页面（展示作者贡献 + 总下载量）
- 增加贡献者排行榜，激励优质创作

## 七、竞争与护城河

### 当前差异化

- 唯一聚焦"Claude Skill + 中国市场"的垂直平台
- 独有的"实践模式"内容形态

### 护城河薄弱点

- 数据来自公开 GitHub，无独占内容
- 社区沉淀几乎为零

### 护城河建设路径

1. **内容护城河**：站内原创评测/教程/对比（独家内容）
2. **社区护城河**：创作者归属感（个人主页 + 粉丝 + 收益分成）
3. **数据护城河**：用户行为数据 → 精准推荐（竞品无法复制）
4. **工具链护城河**：Skill 开发 CLI + 测试工具 + 版本管理
5. **心智护城河**："找 Skill 去 Skill Hub" 的用户习惯

## 八、优先级路线图（两个视角交叉共识）

### P0 — 立即解决

1. **SEO 基础设施**（sitemap + robots + generateMetadata + 首屏 SSR）
2. **移动端详情页展示 Markdown 文档**
3. **社交分享功能**（分享按钮 + OG 图 + Web Share API）
4. **搜索改为客户端实时搜索**

### P1 — 短期优化（1-2 个月）

5. **收藏/书签功能**（利用已有 OAuth）
6. **新用户引导 + 解释 Skill 概念**
7. **展示下载次数**（数据已有，只需前端展示）
8. **标签体系升级**（多标签 + 二级分类）
9. **站内轻量反馈入口**
10. **详情页面包屑导航**

### P2 — 中期建设（3-6 个月）

11. **个人中心**（收藏 / 下载历史 / 我的贡献）
12. **站内投稿表单**（替代 GitHub Issue）
13. **评价/评分系统**
14. **用户行为分析系统**
15. **创作者 Profile + 排行榜**
16. **微信登录支持**

### P3 — 长期规划（6 个月+）

17. 个性化推荐引擎
18. 全文搜索升级
19. 企业版 / API 服务
20. Skill 版本管理与更新通知

---

## 总评

产品在技术实现层面（PC/Mobile 双端适配、骨架屏、无限滚动双保险、GPU 加速、双阶段加载等）做得相当扎实，但在**获客、激活、留存、传播**等增长层面几乎是空白。好消息是改进空间巨大，**SEO + 分享 + 收藏**三件事落地后，就能打开一个全新的增长飞轮。
