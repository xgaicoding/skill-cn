# Skill Hub（MVP）需求文档（PRD）

版本：v0.1.0（MVP）  
日期：2026-01-10  
作者：TBD  

本文档聚焦**产品目标、用户流程、交互规则与验收口径**，不讨论技术方案（账号实现、接口、存储、爬取/同步、安全方案等均不在本文档范围）。

---

## 1. 背景与问题
Claude Code、Codex 等产品已经支持 Skill 生态，但用户在“去哪找/哪些好用/如何快速理解与安装”上缺少统一入口；创作者也缺少统一的曝光与收录渠道。

市面上的 Skill 合集/目录往往“量大但不可用”：收录条目数可能很高，但用户缺乏判断依据（效果如何、是否可复现、是否仍在维护、是否真的能落地），导致“看起来很多、真正能用的很少”。

Skill Hub 目标是打造国内一流的精品严选 Skill 市场：在 MVP 阶段先跑通“发现 → 详情 → 提交 → 收录”的核心路径，并以人工收录的方式满足基础可运营要求。

对用户而言，“知道有哪些 Skill”只是第一步，更关键的是：用户能快速理解**这个 Skill 怎么用、能带来什么效果**。因此，MVP 需要引入“实践（案例/教程）”内容承载，让用户在详情页可直接看到相关实践入口。

---

## 2. 产品定位
### 2.1 一句话定义
Skill Hub 是一个精品严选的 Skill 市场：用户可以浏览 Skill，快速理解“怎么用/效果如何”，并可提交 Skill 收录申请。

### 2.2 MVP 产品目标
- **发现**：用户能通过目录/搜索/标签筛选找到 Skill，并在详情页快速理解用途与用法。
- **登录**：用户支持使用 GitHub OAuth 登录（浏览不强制登录）。
- **提交**：用户能一键跳转到指定的 GitHub Issue（按模板填写信息）提交 Skill 收录申请。
- **收录**：平台维护者基于 Issue 内容人工录入 Skill 条目并公开展示。
- **实践**：每个 Skill 下可挂载多条“实践”，用来展示使用方式与效果，并将用户跳转到实践源链接。
- **热度**：为 Skill 提供“热度”排序能力，帮助用户优先发现更有价值/更常用的 Skill。
- **运营**：通过“人工收录 + 有限入口”的方式，满足基础可运营需求。

### 2.3 非目标（MVP 不做）
- 在线试用/Playground / 在线执行：不做，仅占位。
- 评分/评论体系：不做。
- 收藏/关注等行为体系与榜单：不做（下载次数仅记录，不做下载榜/排序）。
- GitHub 扩展信息展示：不展示语言/License/Commit 列表等；MVP 仅展示 Stars、仓库作者信息与 Skill 更新时间。
- “榜单页”：不做（MVP 仅在首页提供排序能力）。
- 反馈/举报：不做。
- 用户协议/隐私政策/社区规范等说明页面：不做。
- 安全校验（XSS/外链/内容过滤等）：不做。
- 从 `SKILL.md` 自动提取 `name/description/tag`：不做（字段仍由人工录入）。
- 私有仓库/付费内容分发：不做。
- 图标/封面上传与展示：不做。
- 多语言（中英切换）：不做（默认中文）。

## 3. 用户角色与权限
### 3.1 访客（未登录）
- 可浏览目录、搜索、查看详情页。
- 可使用 GitHub OAuth 登录。
- 点击 `提交 Skill` / `提交实践` 需先完成 GitHub 登录。

### 3.2 登录用户（GitHub）
- 可通过 GitHub OAuth 登录/退出登录。
- 可从站内一键跳转到 GitHub Issue 提交 Skill 收录申请或实践提交（Issue 内容按模板填写）。

### 3.3 平台维护者（运营/编辑）
- 负责处理 Issue：确认信息完整性，决定是否收录。
- 负责维护目录内容：新增/编辑/下架/删除 Skill 条目（MVP 不向公众开放站内编辑能力）。
- 维护方式：MVP 不要求提供站内审核/编辑后台；维护者以离线方式维护数据（如脚本/配置/数据库操作均可，技术方案自定）。

## 4. 核心对象定义（产品口径）
### 4.1 Skill（平台收录条目）
Skill 是平台目录中的一个条目，用于描述某个可复用能力（通常对应一个公开仓库/目录中的 Skill 代码与说明）。

#### 4.1.1 Skill 字段（MVP）
平台字段：
- `skill_id`：Skill 唯一标识（系统生成，用于详情页/下载/关联实践等唯一定位）
- `name`：Skill 名称（必填，可重复）
- `description`：简短描述（列表摘要，必填）
- `tag`：标签（必填，枚举，见 4.2）
- `source_url`：来源链接（GitHub 仓库中 Skill 所在目录；仓库根/子目录均可，必填）
- `is_listed`：是否上架（布尔；仅上架 Skill 展示在目录/搜索中；未上架视为“不可见”）

平台统计/计算字段（用于展示/排序）：
- `download_count`：下载次数（累计）
- `practice_count`：实践数量（该 Skill 下已上架的实践条数）
- `heat_score`：热度分（用于热度排序）

GitHub 拉取字段（用于展示/排序）：
- `repo_stars`：仓库 Stars 数
- `repo_owner_name`：仓库作者（Owner）名称
- `repo_owner_avatar_url`：仓库作者（Owner）头像
- `updated_at`：Skill 更新时间（取 `source_url` 对应目录的最新 commit 时间）
- `markdown`：详情页正文（取 `source_url` 对应目录下的 `SKILL.md` 内容）

> 说明：MVP 采用“人工收录”模式，不对公众暴露“提交后可在站内查看状态”的能力；Issue 作为唯一的提交渠道。Source URL 的有效性由维护者在收录时把关。仓库信息（Stars、作者头像/名称/更新时间/SKILL.md）由平台动态拉取并展示。

#### 4.1.2 Skill 热度（MVP）
热度用于衡量一个 Skill 的“被关注程度 + 实践沉淀”，并支持用户按热度对 Skill 列表排序。

热度因子（MVP）：
- 实践数量（`practice_count`，重要）
- 仓库 Stars（`repo_stars`）

分项得分（MVP）：
- `practice_score = practice_count * 1000`（每条实践 +1000）
- `star_score = repo_stars`（每个 Star +1；若尚未拉取成功则为空，页面展示为 `-`）

加权得分（MVP，初始权重可在后续迭代调参）：
- `heat_score = 1.00 * practice_score + 0.15 * star_score`
- 展示：`heat_score` 四舍五入取整（排序使用原始数值）
- 缺失口径：若 `repo_stars` 尚未拉取成功，则 `heat_score` 为空，页面展示为 `-`；目录页排序时将缺失值排在末尾（见 8.1.4）。

> 说明：实践沉淀代表“可复现价值”的强信号，因此贡献最高；Stars 作为辅助参考信号，用于同实践量下拉开差异；更新时间不参与热度，仅用于目录页“最近更新”排序。

> 说明：MVP 只要求“按热度排序的结果稳定且符合直觉”，并支持在页面中展示热度；无需向用户解释热度的具体计算细节。

#### 4.1.3 `markdown`（SKILL.md）内容规则（MVP）
- 详情页正文 `markdown` 来自 `source_url` 对应目录下的 `SKILL.md`（动态拉取）。
- 渲染（MVP）：
  - 默认按 Markdown 渲染展示（标题/段落/列表/代码块/链接等基础能力）。
  - 若检测到内容包含相对路径的图片/链接资源（例如 `./`、`../`、或不以 `http(s)://` 开头的资源链接），则以纯文本方式展示原始 Markdown（不做渲染），并提供 `前往 Source` 入口。
  - 纯文本展示：保留换行与空格（可复制）。
- 缺失与异常：
  - 若未找到 `SKILL.md`：展示“未找到 SKILL.md”，并提供“前往 Source”入口。
  - 若加载失败：展示“内容加载失败”，并提供“重试”入口。
> 说明：MVP 不做安全校验/过滤，仅按基础 Markdown 渲染能力展示。

### 4.2 标签（固定标签集）
- 筛选项：`All`（仅用于筛选，不作为 Skill 的存储值）
- Skill 的 `tag` 枚举（必填，单选）：`研究` / `编程` / `写作` / `数据分析` / `设计` / `生产力`
- 交互规则：仅允许从固定标签集中选择，不支持自由输入；存储为上述中文枚举值

### 4.3 Skill 实践（Practice）
实践是挂载在某个 Skill 下的外部内容，用于帮助用户理解“怎么用”和“效果如何”（例如：教程、案例复盘、真实工作流记录等）。

#### 4.3.1 Practice 字段（MVP）
平台字段：
- `practice_id`：实践唯一标识（系统生成）
- `skill_id`：关联的 Skill（必填）
- `title`：标题（必填）
- `summary`：实践简介（必填，卡片摘要；建议包含使用场景 + 关键效果/结论）
- `channel`：渠道（必填，枚举，见 4.3.2）
- `updated_at`：时间（必填，实践发布时间/更新日期，展示在卡片上；格式 `YYYY-MM-DD`）
- `source_url`：源链接（必填，点击卡片跳转）
- `is_listed`：是否上架（布尔；仅上架实践展示在 Skill 详情页实践区中）
- `is_featured`：是否首页 Banner 推荐（布尔；为 true 则进入 Banner）

平台统计字段：
- `click_count`：点击量（累计）

#### 4.3.2 渠道枚举（MVP）
`飞书文档` / `公众号` / `知乎` / `小红书` / `其他`

#### 4.3.3 点击量口径（MVP）
- 用户点击实践卡片即计：`click_count +1`（不依赖跳转是否成功）
- 点击量仅用于展示与（可选）实践列表排序；MVP 不纳入 Skill 热度计算因子

## 5. 基础规则与声明（MVP）
### 5.1 收录原则（MVP）
- 平台仅收录由维护者确认可公开展示的 Skill 条目。
- 上架与展示：目录页与搜索结果仅展示 `is_listed = true` 的 Skill（下架即 `is_listed = false`）。
- 上架建议门槛（影响力阈值，MVP 初始规则）：`heat_score >= 1.6`（等价：在 `practice_count = 0` 时需 `repo_stars >= 11`；有实践则必达）；维护者可对该门槛做特例处理。
- 自动上下架：MVP 不做“随热度自动上/下架”；`is_listed` 由维护者人工维护（热度阈值仅作为建议）。

### 5.2 免责声明（MVP）
- 平台仅提供信息聚合与展示；Skill 的代码与效果由其作者/来源仓库负责。
- 用户在使用 Skill 前需自行评估使用风险；平台对第三方代码不做担保。

### 6.1 公共页面
1) Skill 目录页（首页）  
2) Skill 详情页  
3) GitHub 登录（OAuth）

5) 提交实践（跳转 GitHub Issue）

## 7. 核心用户路径（MVP）
### 7.1 用户发现路径
目录页 →（搜索/筛选/排序）→ Skill 详情页 →（阅读 `SKILL.md` / 浏览实践卡片）→ 跳转实践源链接或 Source → 离开或继续浏览

### 7.2 用户登录路径（可选）
目录页/详情页 → 点击 `登录` → GitHub OAuth → 返回站内（展示登录态）

### 7.3 用户提交 Skill 路径
目录页/详情页 → 点击 `提交 Skill` → GitHub 登录 → 跳转 GitHub Issue → 按模板填写并提交

### 7.4 用户提交实践路径
Skill 详情页 → 点击 `提交实践` → GitHub 登录 → 跳转 GitHub Issue → 按模板填写并提交

### 7.5 维护者收录与维护路径
查看 Issue → 核对信息完整性 → 人工录入/更新 Skill 与实践 → 动态拉取仓库信息（Stars/作者/更新时间/SKILL.md）→ 对外展示

## 8. 功能需求（详细）

## 8.1 Skill 目录页（首页）
### 8.1.1 页面目标
- 帮助用户快速发现 Skill，并引导进入详情页。
- 默认仅展示“已上架”的 Skill（`is_listed = true`）。

### 8.1.2 页面结构（模块）
- 顶部导航：Logo、搜索框、`提交 Skill` CTA（跳转 GitHub Issue）、`登录/用户头像`（GitHub OAuth）
- Banner（MVP）：左侧站点 Logo + Slogan；右侧运营精选实践轮播（3–6 条，可每周轮换）
- 筛选区：标签 Tabs/Segmented（不使用下拉，单选）、排序下拉
- 内容区：Skill 卡片列表（网格或列表）
- 分页：每页 12 条；支持 Prev/Next + 页码
- Footer：免责声明

### 8.1.3 Skill 卡片信息结构（MVP 必须）
- 名称（点击进入详情，按 `skill_id` 路由）
- 简短描述（1–2 行截断）
- 标签（Badge，单个展示）
- Stars 数（仓库 Stars）
- 仓库作者头像 + 作者名称
- 实践数量（`practice_count`）
- 更新时间（可选展示）
- 热度（`heat_score`）

### 8.1.4 筛选与排序
- 排序（MVP）：
  - `热度最高`：按 `heat_score` 倒序（`heat_score` 为空排末尾；同分按 `updated_at` 倒序 → `repo_stars` 倒序 → `skill_id` 升序）
  - `最近更新`：按 `updated_at` 倒序（`updated_at` 为空排末尾；同时间按 `heat_score` 倒序 → `repo_stars` 倒序 → `skill_id` 升序）
- 默认排序：`热度最高`
- 标签：All + 固定标签集（见 4.2），单选，展示为 Tabs/Segmented（不使用下拉）

### 8.1.5 搜索
- 入口：顶部全局搜索框
- 范围：`name` + `description` + `tag`
- 结果范围：仅在 `is_listed = true` 的 Skill 集合内搜索与返回结果
- 交互：回车触发；支持清空搜索回到默认列表

### 8.1.6 状态与空态
- 加载中：骨架屏/占位
- 无结果：提示“暂无结果”，并提供“清空筛选/清空搜索”入口

### 8.1.7 GitHub 信息展示规则（MVP）
- Stars：
  - 展示为整数；≥1000 可展示为 `1.2k` 形式（规则在视觉稿中统一）。
  - 若尚未拉取成功：展示占位 `-`（不阻塞列表展示）。
  - 时效：页面刷新后应展示 GitHub 最新 Stars 数（允许存在极短的同步延迟）。
- 作者信息（头像/名称）：
  - 名称展示优先级：GitHub 展示名（Display name）> GitHub 用户名（login）。
  - 头像加载失败：展示默认头像占位。
- 更新时间：
  - 展示 `updated_at`（Skill 目录最新 commit 时间）。
  - 若尚未拉取成功：展示占位 `-`（不阻塞列表展示；“最近更新”排序时缺失值排在末尾，见 8.1.4）。

### 8.1.8 运营精选实践推荐（Banner，MVP）
- 目标：让用户在首页优先看到“可复现的实践”，快速判断 Skill 价值与用法。
- 结构：左右分栏
  - 左侧：站点 Logo + Slogan（品牌区，静态）
  - 右侧：运营精选实践轮播（3–6 条）
- 运营机制（MVP）：运营手动精选 3–6 条实践，建议每周轮换一次（无自动推荐算法）。
- 展示与筛选无关：Banner 展示不受标签筛选/搜索词影响（筛选/搜索仅作用于下方 Skill 列表）。
- 展示范围：仅展示 `is_listed = true` 的 Practice，且其所属 Skill 也需 `is_listed = true`。
- 数量：3–6 条（不足 3 条则不展示该 Banner）。
- 排序：按 `updated_at` 倒序（仅展示 `is_featured = true`）。
- 卡片展示字段：实践标题、实践简介、渠道、时间（`updated_at`）、关联 Skill 名称（可选展示标签）。
- 交互：
  - 轮播：支持左右切换（箭头）+ 位置指示（圆点）；默认自动轮播（每 6 秒切换一次），用户手动切换后重新计时。
  - 点击实践卡片：点击即计 `click_count +1`（不依赖跳转是否成功），并跳转到实践 `source_url`（新开窗口）。
  - 可选：提供“查看 Skill”入口，跳转到 Skill 详情页。

## 8.2 Skill 详情页（MVP）
### 8.2.1 页面目标
- 让用户快速确认：这个 Skill 做什么、怎么用、效果如何（通过实践）、来自哪里。

### 8.2.2 页面模块（MVP）
- 标题区：名称、标签、简短描述
- 仓库信息区：Stars 数、仓库作者头像、仓库作者名称、`Source（链接）`
- 操作区：
  - `下载 ZIP`（一键下载当前 Skill 目录的 ZIP 包）
  - `Run`（置灰/不可用）+ 文案 `敬请期待`
- 基础信息区：`skill_id`（可复制）、`更新时间`、`热度`
- 实践区：实践卡片列表 + `提交实践` CTA
- 内容区：`SKILL.md`（默认按 Markdown 渲染；若包含相对资源链接则以纯文本展示）

### 8.2.3 一键下载 ZIP（MVP）
- 按钮文案：`下载 ZIP`
- 下载内容口径（MVP）：下载该 Skill 对应的“目录内容”ZIP 包（以 `source_url` 指向的目录为准）。
- 若 `source_url` 指向仓库根目录：下载内容等同于整个仓库。
- ZIP 内容口径：仅包含该目录下的文件与子目录（ZIP 根目录以 Skill 目录为根）。
- 下载次数：用户点击 `下载 ZIP` 按钮即计（不依赖下载是否成功），该 Skill `download_count +1`。
- 异常：
  - 若 `source_url` 不合法或无法下载：提示“下载失败”，并提供“前往 Source”兜底入口。

### 8.2.4 异常与错误提示（MVP 必须覆盖）
- 内容加载失败：提示“内容加载失败”，提供“重试”按钮
- 条目不存在/已下架/已删除：提示“该 Skill 不存在或已被移除”，引导回首页

### 8.2.5 实践（MVP）
#### 8.2.5.1 展示规则
- 展示该 Skill 关联的已上架实践卡片（`practice_count`）。
- 默认排序：按 `updated_at` 倒序（最新在前）。

#### 8.2.5.2 实践卡片封面字段（MVP）
- 标题（`title`）
- 实践简介（`summary`，1–2 行截断）
- 时间（`updated_at`）
- 渠道（`channel`）
- 点击量（`click_count`）

#### 8.2.5.3 交互（MVP）
- 点击卡片：点击即计 `click_count +1`（不依赖跳转是否成功），并跳转到 `source_url`（新开窗口）。
- `提交实践`：跳转到 GitHub Issue（实践模板）；需登录。

#### 8.2.5.4 空态
- 若暂无实践：展示“暂无实践”，并提供 `提交实践` CTA。

## 8.4 提交 Skill（GitHub Issue，MVP）
### 8.4.1 入口
- 目录页 Header 提供 `提交 Skill` CTA
- 任意页面 Header 均可复用该入口
- 跳转到同一个 GitHub Issues 页面（由运营维护）

### 8.4.2 Issue 模板（MVP）
提交 Skill 的 Issue 模板需包含以下字段（用户按模板填写）：
- Source URL（必填，建议填写 GitHub 仓库/目录链接；若 Skill 在子目录，请填写该目录链接）
- 名称（必填，展示在卡片与详情页）
- 简短描述（必填，展示在卡片与详情页）
- 标签（必填，从固定标签集中选择一个）
- `SKILL.md`：请确认该目录下存在 `SKILL.md`（平台将直接展示其内容）
- 其他补充（可选：截图/演示链接/适用场景）

> 说明：平台会基于 Source URL 动态拉取并展示仓库信息（Stars、作者头像/名称、更新时间、SKILL.md）。

### 8.4.3 提交后提示（MVP）
- 跳转到 GitHub 后，用户提交 Issue 即视为“已提交收录申请”
- 平台站内不展示“审核中/已拒绝”等状态（由维护者在 GitHub Issue 中沟通与关闭）
- 处理 SLA：承诺在 3 天内处理（通过评论/关闭 Issue 的方式告知结果）

## 8.5 提交实践（GitHub Issue，MVP）
### 8.5.1 入口

### 8.5.2 Issue 模板（MVP）

- 标题（必填）

### 8.5.3 提交后提示（MVP）

- 处理 SLA：承诺在 3 天内处理（通过评论/关闭 Issue 的方式告知结果）

## 9. 时间与展示口径
- 时区：时间展示与“最近更新”口径均按北京时间（UTC+8）。
- 展示格式：页面中时间统一展示为 `YYYY-MM-DD`。
- 目录页排序：
  - 热度最高：按 `heat_score` 倒序
  - 最近更新：按 `updated_at` 倒序
- 详情页展示“更新时间”：显示 `updated_at`（取 Skill 目录最新 commit 时间）
- Stars/作者信息/更新时间：以 GitHub 当前值为准，刷新页面后应展示最新数据
- 实践时间：展示 `updated_at`
- 热度：展示 `heat_score`

## 11. MVP 交付清单（验收口径）
- 登录：支持 GitHub OAuth 登录/退出；Header 展示登录态（登录按钮/用户头像）
- 目录页：仅展示 `is_listed = true` 的 Skill（上架口径见 5.1）；可按标签筛选、可搜索、可排序（热度最高/最近更新）、可分页（12 条/页）、可进入详情；卡片展示 name/description/tag、热度、Stars、作者头像/名称、实践数量、更新时间（可选）
- 首页 Banner：展示“运营精选实践推荐”（3–6 条，按 `updated_at` 倒序；口径见 8.1.8）
- 详情页：展示 name/description/tag/source、热度、Stars、作者头像/名称、更新时间；支持 `下载 ZIP`（下载当前 Skill 目录 ZIP，并记录下载次数）；展示 `SKILL.md`；展示实践卡片列表（标题/简介/时间/渠道/点击量）与 `提交实践` CTA
- 提交入口：
  - `提交 Skill`：需登录后跳转到 GitHub Issue（按模板提交）；承诺 3 天内处理
  - `提交实践`：需登录后跳转到 GitHub Issue（按模板提交）；承诺 3 天内处理
- 内容维护：维护者可人工录入/更新/下架/删除 Skill（站内不开放公众编辑）
- 声明：至少提供免责声明露出

## 12. 已确认决策（MVP）
1) `提交 Skill` / `提交实践` 需要先完成站内 GitHub 登录；MVP 不做反馈与举报。  
2) Source URL 不做自动校验，由维护者在收录时人工校验。  
3) 时间口径：仅保留“更新时间”，取 Skill 目录最新 commit 时间；支持“热度最高/最近更新”排序（默认：热度最高）；不做时间筛选。  
4) GitHub 信息：Stars、作者信息、更新时间、`SKILL.md` 均动态拉取并展示最新数据。  
5) 实践：实践卡片展示在 Skill 详情页；实践点击跳转源链接并记录 `click_count`；实践通过 Issue 提交并由维护者录入挂载到 Skill。  
6) 热度：`heat_score` 由 Stars、实践数量加权得到（实践数量权重最高），用于首页排序，并在页面中展示热度。  
7) `下载 ZIP`：下载当前 Skill 目录的 ZIP 包（非整仓库），并记录 `download_count`（点击即计）。  
8) 处理 SLA：提交 Skill / 提交实践均承诺 3 天内处理。  
9) 唯一标识：Skill 使用 `skill_id`；实践使用 `practice_id`；Skill `name` 允许重复。  
10) 用户协议/隐私政策/社区规范说明页：MVP 暂不需要。  
11) 安全校验：MVP 不做。  
12) 上架与门槛：目录仅展示 `is_listed = true`；默认上架建议门槛为 `heat_score >= 1.6`（热度计算见 4.1.2）；维护者可特例处理。
